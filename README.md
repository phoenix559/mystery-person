🕵️‍♂️ Mystery Person Game (ESL Activity)

An interactive, web-based speaking game designed for ESL (English as a Second Language) students to practice using the Simple Present of the verb "Be" (am, is, are).

🎯 Grammar Focus

This activity trains students to ask for and provide personal information using only the verb "be". It discourages the use of "do/does" and action verbs, forcing students to master:

Identity: Who is he? Are you an actor?

Origin/Location: Where are you from? Where is she now?

Age: How old are you?

Description/Status: Are you married? Is he alive?

👥 How to Play

This game is designed for pair-work or small groups in both physical and Hyflex classrooms.

Partner A (The Mystery Person): Clicks the "Next Mystery Person" button to generate a secret identity card. They must hide their screen from Partner B.

Partner B (The Interviewer): Asks "Wh-" and "Yes/No" questions using the verb "be" to deduce who Partner A is.

The Rules: Partner A can only answer with "Yes, I am," "No, I'm not," or full sentences using the verb "be".

Example Questions to Ask:

Are you male or female?

Are you alive?

Where are you from?

What is your job?

How old are you?

Who is your spouse?

✨ Features

50 Notable Figures: Includes a diverse mix of prominent Hispanic artists/athletes and massive international stars to keep students highly engaged.

Dynamic Age Calculation: Automatically calculates the person's exact current age based on their Date of Birth (DOB) and today's date.

Live Wikipedia Avatars: Pulls current thumbnail images directly from Wikimedia Commons. Includes an automatic fallback to an initial-based avatar if an image link ever breaks.

Single-File Architecture: The entire app (HTML, CSS, JS, and Data) is contained in one file, making it incredibly easy to host anywhere or embed directly into an LMS like Canvas or Google Sites.

🛠️ Setup & Hosting

Because this is a single index.html file, it requires no server or database to run.

Run Locally: Just double-click index.html to open it in any browser.

Host on GitHub Pages: Upload to a public repository and enable GitHub Pages in the settings.

Embed in Canvas / Google Sites: Copy the raw HTML code and paste it into any standard HTML <embed> or <iframe> block.
